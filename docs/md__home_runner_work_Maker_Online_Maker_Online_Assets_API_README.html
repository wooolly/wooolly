Maker.API

The Maker.API is the primary component between Maker: Online and player-created gamemodes. There's various implementation details which are important to understand the reasoning behind, hence this README file for the API.

Maker Event Functions

A "Maker Event Function" is any method inside a base class of Maker.API, which is non-public and invoked by Maker: Online. They're equivalent to Unity's event functions.

The reason they're non-public and not overridable (for the most part) is for a few reasons... 1) Fail safe: They shouldn't be called by anything other than Maker: Online, so it saves the developer from themselves. 2) Optimisation: Maker: Online preloads all event functions, and those that aren't present will not be loaded and a call will not be attempted. With an interface that forces implementation, or a base class which provides virtual methods, this optimisation wouldn't be possible because the methods would always be loaded and always be invoked. It's a minor optimisation. 3) Dependency Injection: Event functions support dependency injection, thus, by having no interface to conform to, we can have any parameters we desire as the gamemode developer, and they'll be injected.

Constructors

To create a new object that represents something within the game, Maker requires a constructor. Constructors are a little different to methods in C#, and cannot be inherited, but we don't want to force gamemode developers to implement a specific constructor each time. Therefore, we have a special event function called OnInit for most base classes (exception is Gamemode) which gets called before any other, and is designed to be a replacement for using a constructor. By using a constructor, there's no guarantee that everything will be initialised at that point, so always do setup after calling base.OnInit() within the override.

Note I: Gamemode is a little different to most other base classes.

Gamemodes are instantiated before caching and other important processes have occurred, hence why they're so different.

Note II: Gamemodes are abstract since the gamemode should have at least

one and there can't be a default.
