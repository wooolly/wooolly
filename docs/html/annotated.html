 CGamemodeThe abstract base gamemode class is the root of any type that controls the overall functioning of the hosted gamemode. This is the primary starting point when building your gamemode. There can be any number of gamemode classes, and at least one gamemode class must be declared. Plugins will usually provide their own gamemode class so that they can perform their own setup. Plugin gamemodes are always initialised before the core gamemode implementations. A single instance of each gamemode type will be instantiated once the DLL's are loaded and the server is about to start accepting connections
 CPlayerA base class which represents a Maker: Online player. This base class is used by default when a player joins the server, until a different type is specified. It's implementation is minimal, so if this default type is used, nothing will happen with the player once they have connected. This type is only here to allow players to connect even when there isn't a proper implementation of player. ideal for your initial testing when you have declared a gamemode class and nothing else
